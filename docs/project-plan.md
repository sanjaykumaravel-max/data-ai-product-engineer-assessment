# Project Plan (Living Document)

## Objective
Deliver both assessment tasks in 4 days with clear reasoning, practical scope, and professional communication.

## Constraints
- Single repository for both tasks
- 4-day total timeline
- Reasoning and explainability prioritized over complexity

## Scope Strategy
- Build only what is required to satisfy each task clearly.
- Treat optional complexity as out of scope unless it improves explanation quality.
- Keep each task implementation demoable and easy to review.

## Decision Log Template
Use this format for all major decisions:

- Date:
- Task:
- Decision:
- Alternatives considered:
- Why this choice:
- Tradeoff accepted:

## Risks and Mitigations
- Risk: overengineering
  - Mitigation: define MVP per task before coding
- Risk: weak walkthrough narrative
  - Mitigation: keep `docs/walkthrough-script.md` updated continuously
- Risk: unclear progression
  - Mitigation: small regular commits tied to specific milestones

## Daily Milestones
- Day 1: setup + scope + assumptions + initial skeleton
- Day 2: Task 1 core build + notes
- Day 3: Task 2 core build + notes
- Day 4: polish + tests + walkthrough capture
